## Implementation Notes

### Data Sources Needed:
1. **Color-Object mappings**: 200+ common objects mapped to their typical colors
2. **Shape-Object mappings**: 150+ common objects mapped to their shapes
3. **Geometric shape data**: Properties (sides, angles, vertices) for 2D and 3D shapes
4. **Pattern templates**: Various sequence types with generation rules
5. **Semantic templates**: 8-12 templates per sub-type

### Generation Strategies:
- **1A, 2A**: Random selection from color/shape-object mappings × templates
- **1B, 2B**: Random selection with consistent single answers per query
- **1C, 2C**: Balanced 70/30 true/false distribution
- **3A-3L**: Systematic enumeration with template variation
- **4A-4H**: Rule-based generation with randomized starting points

### Validation Checks:
- All color-object associations are factually correct
- All shape-object associations are reasonable
- All geometric calculations are mathematically verified
- Pattern sequences follow stated rules
- No duplicate queries

### Answer Format Rules:
- Colors: lowercase single word (e.g., "yellow", "red")
- Shapes: lowercase shape name (e.g., "circle", "sphere", "rectangle")
- Numbers: numeric string (e.g., "3", "12", "25")
- Yes/No: lowercase (e.g., "yes", "no")
- Patterns: exact next item (e.g., "blue", "5", "Thursday")
